# 单词文档配音—完整方案

## 一、项目概述

“单词文档配音”面向拥有静态词汇 PDF、但缺少配套发音的语言学习者。软件把规则词表提取成结构化词条，为每条内容生成独立语音，并支持搜索、校对、逐词播放和 Anki 导出。

它不是从第一页连续朗读到最后一页的普通 PDF 阅读器。核心要求是让“序号、单词、注音、释义、页码、异常标记和音频”始终保持一一对应。

## 二、当前 v0.6.0 范围

已确认的目标输入包括：

- 规则四列表格：`序号 | 单词 | 注音 | 释义`，107 页、4450 条、序号连续；
- 双栏中英词汇笔记：恢复左右栏阅读顺序，生成可校对候选；
- 扫描型 PDF：识别失败时提示安装可选 OCR 扩展，不上传原文。

v0.6.0 提供：

1. 只读分析 PDF 并提取完整词表；
2. 标记缺少注音、重复拼写、多读音和可疑源数据；
3. 搜索、筛选、编辑和逐词试听；
4. 使用本地 Kokoro ONNX 生成英语 WAV；
5. 先生成 30 词样本，确认后可生成全部；
6. 用“只看已有音频”检查当前已完成内容，并可直接打开 WAV 文件夹；
7. 在全部生成前先导出已有音频的 Anki `.apkg`，全部完成后再严格导出全量卡组；
8. 提供无需系统 Python 的 Windows 便携版；
9. 声音或语速改变后自动重新生成不匹配的旧缓存，并显示中文友好声音名称；
10. 按数字对序号进行升序、降序排列。
11. 原创治愈风界面，五枚原创内置贴纸与用户本地贴纸随机轮换，包含 12 条非连续重复的温暖短句；
12. 适配 Windows 150% 显示缩放，首屏完整显示摘要、设置、词表和所有操作。
13. 珊瑚粉可爱小书程序图标，以及升降序一致的稳定表格列宽。
14. 启动后由用户从已导入项目或新 PDF 中选择词汇，不再自动分析固定文档。
15. 从任意选中行开始连续播放，单词重复 1—5 次、词间暂停 0—10 秒，并支持英文或英文＋Windows 离线中文释义。
16. 把词条标记为认识、模糊或不认识，重点复习自动筛选模糊和不认识。
17. 数据库迁移前备份、每日自动备份、手动备份和完整性检查后的恢复；继续兼容 v0.2—v0.3 数据。
18. 用户可以添加、轮换和删除自己的贴纸，个人图片只保存在本机，不上传或进入安装包。
19. 专注学习版主界面：紧凑标题栏、单行筛选、四列词表和常驻底部播放器。
20. 将音频生成、Anki、编辑、备份、贴纸等低频操作收纳到更多菜单，将播放参数集中到学习设置，原功能保持可访问。
21. 上一词和下一词使用无边框传输图标，中间连续播放使用固定圆形珊瑚色主按钮。
22. 自动区分规则词表与双栏笔记，复杂文字层 PDF 使用坐标分栏解析。
23. 复杂候选保存页码、原始边界框、解析方式和置信度。
24. 导入预览支持编辑、勾选、删除、合并和高置信度筛选，确认前不写数据库。
25. 后台分析显示逐页进度并支持安全取消。
26. v0.6.0 数据库迁移前自动备份，旧音频、人工编辑和学习状态直接兼容。
27. 扫描件保持 OCR 可选扩展边界，基础安装包不捆绑大型模型。

## 三、核心流程

```text
点击“选择词汇”
  → 打开已导入项目，或选择新的 PDF
  → 自动判断规则表格或复杂双栏版面
  → 后台逐页解析，可随时取消
  → 复杂结果先预览、编辑和确认
  → 校验词条与异常数据
  → 写入本地 SQLite
  → 搜索 / 异常筛选 / 重点复习 / 编辑 / 从所选开始连续播放
  → 标记认识 / 模糊 / 不认识
  → 生成 30 词样本或生成全部
  → 导出已有音频，或完成后导出全量 Anki 卡组
```

## 四、技术结构

| 模块 | 技术 | 职责 |
|---|---|---|
| 桌面界面 | Qt / PySide6 Essentials | 非阻塞导入、词表、编辑、进度和操作反馈 |
| PDF 提取 | pdfplumber | 按页面表格提取四列数据 |
| 本地数据 | SQLite | 原始字段、人工修改、异常、音频状态、学习状态和版本迁移 |
| 本地发音 | kokoro-onnx | 离线生成 WAV，按词条稳定缓存 |
| 中文释义 | Windows System.Speech | 使用系统离线中文语音；缺失时安全降级 |
| 音素转换 | eSpeak NG | 为 Kokoro 提供英语音素，兼容 Windows 中文路径 |
| Anki 导出 | genanki | 稳定卡片 ID 和实际媒体打包 |
| Windows 成品 | PyInstaller | 便携目录版，内置模型和运行库 |

正式代码位于 `src/word_voice/`：

- `extractor.py`：PDF 提取和结构校验；
- `storage.py`：SQLite、学习状态、自动备份、迁移与恢复；
- `speech.py`：中文释义清理和 Windows 离线朗读；
- `tts.py`：本地发音、缓存和文件命名；
- `anki_export.py`：卡片与媒体打包；
- `app.py`：桌面界面和后台任务；
- `cli.py`：命令行入口。

## 五、关键产品决策

- 原始 PDF 永远只读，不覆盖、不提交到 Git。
- 使用 PDF 序号作为词条唯一标识；同形词不能共用音频。
- 音频文件名包含序号，例如两个 `present` 仍对应不同文件。
- 人工修改保留原字段，重新导入时不得静默覆盖。
- 生成失败必须可见、可重试；有效缓存不重复生成。
- 本地处理优先，不上传 PDF、词表或音频。
- 公开版只使用原创贴纸；用户自定义图片只保存在本地数据目录。
- 任何数据库字段迁移和恢复前先建立一致性备份。

## 六、已知源数据异常

- 序号 583 的 `a.m` 缺少注音。
- `present` 出现两次，词性与读音不同。
- `forbid` 出现两次，其中一条疑似应为 `forbidden`，保留源数据并提示人工确认。
- 部分词条含多个读音，默认采用主要读音并进入异常筛选。

- 30 个代表词已生成有效的 24 kHz 单声道 WAV；

- v0.3.0 源码界面在 150% 显示缩放下完整加载 107 页、4450 条、89 条需留意和 1104 条已有音频。
