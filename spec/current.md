# 当前版本规格（v0.1 英文词表点读原型）

状态：可试听验收（完整 4450 词音频等待首批 30 词确认）  
最后更新：2026-07-17

## 一句话目标

把目标英语四级词汇 PDF 稳定转换为可逐词播放、可校对并可导出到 Anki 的本地学习资料。

## 已确认输入

- 文档：`英语四级词汇完整版带音标-乱序版--共107页.pdf`
- 类型：文本型 PDF，Excel 导出的规则表格，不需要 OCR。
- 结构：`序号 | 单词 | 注音 | 释义`。
- 数据量：107 页、4450 条、序号 1-4450 连续。

## 当前范围

1. 从目标 PDF 提取全部 4450 条词汇和来源页码。
2. 使用序号作为稳定唯一标识，保留重复拼写词条。
3. 标记缺失注音、重复拼写、多读音、可疑单词和生成失败。
4. 允许用户查看、搜索、试听并修改结构化词条。
5. 使用本地 Kokoro ONNX 生成美式或英式英语单词发音。
6. 音频按词条缓存，支持中断后继续和失败重试。
7. 导出包含实际音频的 Anki `.apkg` 卡组。
8. 在 Windows 本地提供可运行的桌面界面。

## 关键产品决定

- 原始 PDF 始终只读，不复制到仓库，不覆盖源文件。
- 第一版只保证目标 PDF 和同类四列表格，不加入扫描件 OCR。
- 发音引擎与文档提取解耦，第一版默认本地 Kokoro ONNX。
- 默认美式英语，界面保留英式英语声音选择。
- 先生成 30 个代表性词的试听样本，确认后再批量生成 4450 条。
- 同形词不共用缓存键；音频文件名包含 PDF 序号。
- 人工修改保留原始字段，重新导入时不静默覆盖人工修改。

## 第一版桌面流程

```text
选择 PDF
  -> 提取四列表格
  -> 显示 4450 条词汇与异常统计
  -> 搜索/筛选/编辑/逐词试听
  -> 生成 30 词样本或生成全部
  -> 导出包含音频的 Anki 卡组
```

## 交付物

1. 可重复运行的 PDF 提取入口。
2. CSV、JSON 和 SQLite 结构化结果。
3. 30 个代表性单词的本地发音样本流程。
4. 可逐词播放和检查异常的 Windows 桌面界面。
5. 包含音频的 Anki `.apkg` 导出功能。
6. 针对目标 PDF 的自动验证和测试记录。
7. 安装、模型准备和使用说明。

## 验收条件

- 精确提取 4450 条记录，最小序号为 1、最大序号为 4450，无缺失和重复序号。
- 原始 PDF 的内容、时间戳和文件哈希在处理前后保持不变。
- 每条记录可以独立播放，不因拼写相同复用错误音频。
- 生成失败可见、可重试，重启后不重新生成有效缓存。
- 人工修改在重启和重新导入后仍然保留。
- Anki 卡组可导入，随机抽查至少 50 条时字段和音频对应正确。

## 已知源数据异常

- 序号 583：`a.m` 缺少注音。
- `present` 出现两次，注音和词性不同，必须保留为独立词条。
- `forbid` 出现两次，其中一条释义和注音疑似对应 `forbidden`，需要人工确认。
- 部分词条在同一注音单元中包含多个读音，第一版默认读主要读音并进入异常筛选。

## 暂不纳入 v0.1

- 扫描 PDF OCR 和任意复杂版式。
- 直接在原 PDF 中嵌入音频。
- 手机独立应用、账户、云端同步和多人协作。
- 例句生成、例句配音、声音克隆和视频配音。
- 自动纠正源教材中的全部拼写、音标和释义问题。
