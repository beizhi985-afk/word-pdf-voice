# 当前版本规格（v0.2.1 语音设置与序号排序）

状态：开发完成，便携版已验证并部署
最后更新：2026-07-18

## 一句话目标

修复声音和语速被旧缓存遮蔽的问题，让设置变化后真正重新生成音频，并提供明确的序号升序、降序功能。

## 已确认输入

- 文档：`英语四级词汇完整版带音标-乱序版--共107页.pdf`
- 类型：文本型 PDF，Excel 导出的规则表格，不需要 OCR。
- 结构：`序号 | 单词 | 注音 | 释义`。
- 数据量：107 页、4450 条、序号 1—4450 连续。

## v0.2.1 修复范围

1. 音频缓存记录声音、语速和语言配置；配置变化后，下一次试听或批量生成自动重新制作对应 WAV。
2. v0.2.0 的默认 Sarah 0.90 倍速音频可原样复用，并自动补写缓存配置。
3. 声音下拉框显示中文口音、性别和名称，不再只显示内部声音代码。
4. 语速显示“倍”单位，并在变化后给出明确生效提示。
5. 增加“序号从小到大”和“序号从大到小”，序号按整数而不是文字排序。
6. 数据库自动增加 `audio_profile` 字段，继续使用现有 v0.2 数据目录，不复制 1104 个 WAV。

## v0.2.0 新增范围

1. 工具栏增加“只看已有音频”，可与搜索和“只看异常”组合使用。
2. 操作区增加“打开音频文件夹”，直接查看当前项目的 WAV 文件。
3. 增加“导出已有音频”，只打包状态有效且文件存在的词条。
4. 保留“导出全部 Anki”的严格校验；未完成全部音频时仍拒绝正式全量导出。
5. 摘要区在试听或批量生成结束后刷新已有音频数量。
6. v0.2.0 使用独立程序目录和独立数据目录；首次打开同一 PDF 时，从 v0.1 复制一次数据库快照与有效音频。
7. 从 v0.2.1 起采用普通升级，不再为每个版本复制整套用户数据；提升版本号并完整维护更新日志。
8. Anki 导出对教材中的 `<古>`、`<口>` 等尖括号文字进行安全转义。

## 保留能力

- 提取全部 4450 条词汇和来源页码。
- 序号作为稳定唯一标识，重复拼写保持独立。
- 异常标记、搜索、编辑、逐词试听和音频缓存。
- 本地 Kokoro ONNX 美式或英式发音。
- 中断续做、失败可见和有效音频跳过。
- 导出包含真实 WAV 的 Anki `.apkg`。
- Windows 便携版，不依赖系统 Python。

## 用户流程

```text
选择 PDF
  → 首次进入 v0.2 时复制 v0.1 已有结果
  → 查看全部词条或勾选“只看已有音频”
  → 试听 / 编辑 / 继续生成
  → 可随时“导出已有音频”验证 Anki
  → 全部完成后“导出全部 Anki”
```

## 版本与数据边界

- v0.1.1 程序和 `%LOCALAPPDATA%\WordPdfVoice\projects\` 保持不变。
- v0.2.1 程序目录为 `dist\WordPdfVoice-v0.2.1\`，与 v0.2.0 共用 v0.2 数据目录。
- v0.2.0 数据目录为 `%LOCALAPPDATA%\WordPdfVoice\v0.2\projects\`。
- 首次迁移使用 SQLite 一致性快照；每个已完成音频复制到 v0.2 目录并重写数据库路径。
- 迁移完成后，删除或继续运行 v0.1 都不会影响 v0.2 已复制的数据。

## 验收条件

- 勾选“只看已有音频”后，只显示数据库状态为 `ready` 的词条。
- 搜索、异常筛选和已有音频筛选可组合使用。
- 部分 Anki 导出只包含实际存在的音频，不因其余词条未生成而失败。
- 没有任何有效音频时，部分导出给出明确提示，不生成空卡组。
- 全量导出仍要求全部词条有有效音频。
- v0.2 首次迁移后，数据库、人工编辑和音频路径均指向 v0.2 独立目录。
- 真实 PDF 仍提取 4450 条、序号连续，原始 PDF 哈希不变。
- 便携版必须通过真实 24 kHz 单声道 WAV 生成门禁。
- Anki 数据库中尖括号教材标记按文字保留，不被解释为 HTML 标签。
- 相同声音和语速重复试听时复用缓存；声音或语速变化后重新生成。
- v0.2.0 默认音频首次由 v0.2.1 使用时不重复生成，只补写配置。
- 序号升序为 `1、2、3…`，降序为 `4450、4449、4448…`，不能按字符串排序。
- 便携版真实生成不同声音以及 0.65、1.25 倍速 WAV，并验证声音内容和音频长度确实不同。
