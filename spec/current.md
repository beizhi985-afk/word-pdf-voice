# 当前版本规格（v0.6.0 智能文档解析版）

状态：开发完成并通过正式包真实 PDF 与发音门禁
最后更新：2026-07-22

## 一句话目标

在不破坏既有词库和音频的前提下，让规则词表与双栏中英笔记都能安全导入；复杂结果必须先预览确认，再写入词库。

## v0.6.0 更新范围

1. 导入时先识别文档类型：规则四列表格继续走原有快速路径，未发现表格结构时改走坐标分栏解析。
2. 双栏解析按“每页左栏从上到下，再到右栏从上到下”的顺序配对英文和中文，保存来源页码、文本坐标、识别方式与置信度。
3. 复杂文档结果不直接写入数据库，先进入预览窗口；用户可以编辑字段、取消勾选、删除或合并行，确认后才加入词库。
4. 低密度说明页自动跳过并记录原因；低置信度内容明确标记，手写图形不冒充可靠音标。
5. 分析继续在 Qt 后台线程运行，显示逐页进度，支持安全取消；取消预览或取消分析不会改变现有词库。
6. 数据库迁移到 v0.6.0 前自动备份，新增置信度、来源坐标和识别方式字段；已有词条、音频、人工编辑和学习状态保持兼容。
7. 没有文字层且无法形成中英配对时，明确提示使用 OCR 扩展；基础安装包不捆绑大型 OCR 运行时。
8. 源码、`pyproject.toml`、PyInstaller、构建目录和可执行文件统一使用 v0.6.0。

## v0.6.0 验收条件

- 《雅思阅读笔记.pdf》识别为复杂版面，跳过第 1 页说明内容，得到 1772 条可预览候选，页码和左右栏顺序正确。
- 原四六级 PDF 仍提取 107 页、4450 条，序号 1—4450 连续，首末词和已知边界项目不变化。
- 复杂文档在用户确认前不写入项目数据库；取消分析或取消预览后主程序可继续使用。
- v0.5.0 数据库首次由 v0.6.0 打开时先生成完整性备份，再增加新字段；旧音频路径和学习状态不变化。
- 版本一致性检查、自动测试、便携版构建、真实 PDF 回归和关键界面截图全部通过。

## v0.5.0 更新范围

1. 主界面采用紧凑标题栏、单行筛选区、四列词表和常驻底部播放器，与用户确认的专注学习版示例图保持同一信息层级。
2. 顶部只保留“选择词汇”和更多菜单；音频生成、Anki 导出、编辑、备份恢复、贴纸和短句按用途收纳，功能不删除。
3. 词表只常显序号、单词、音标和中文释义；页码、异常、音频和熟悉度状态通过筛选、行提示及播放器保留。
4. 声音、语速、重复 1—5 次、词间暂停和播放内容集中到“学习设置”对话框。
5. 底部播放器跟随选中词条，提供试听、上一词、圆形连续播放键、下一词和认识／模糊／不认识标记。
6. 中间播放键必须是 60×60 圆形珊瑚色按钮；上一词和下一词必须是 44×44 无边框传输图标，不得退化为普通方形按钮。
7. v0.5.0 不改变数据库结构，继续使用 v0.4.0 数据结构和 v0.2 数据目录，旧词条、音频、标记、贴纸和备份直接兼容。
8. 源码、构建配置、构建目录和可执行文件统一使用 v0.5.0。

## 已确认输入

- 结构：`序号 | 单词 | 注音 | 释义`。
- 数据量：107 页、4450 条、序号 1—4450 连续。

## v0.4.0 更新范围

1. 从当前表格选中行开始，按当前排序和筛选顺序自动播放后续词条。
2. 单词重复次数可选 1—5 次，词条间暂停可选 0—10 秒。
3. 播放内容可选“只听英文”或“英文＋中文释义”；英文继续使用 Kokoro 离线音频，中文优先使用 Windows 离线中文语音，无可用中文语音时提示并降级。
4. 每个词条可标记为未标记、认识、模糊或不认识；重点复习只显示模糊和不认识。
5. 顶部增加换贴纸和用户自定义贴纸管理；个人图片只保存在本机 `%LOCALAPPDATA%\WordPdfVoice\v0.2\custom-stickers\`。
6. 公开版删除商业动漫角色贴纸，内置学习者、云朵读书猫、月兔、橘狐和学习机器人五枚原创透明贴纸。
7. 旧数据库自动增加 `learning_status`，迁移前建立一致性备份；词条、人工修改、音频缓存和配置保持兼容。
8. 每天首次打开、重新导入和恢复前自动备份；选择词汇窗口提供立即备份、历史列表和完整性检查后的恢复。
9. 源码、`pyproject.toml`、PyInstaller、构建目录和可执行文件统一为 v0.4.0，构建前自动检查版本一致性。

5. 选择窗口提供“导入新 PDF”；没有历史项目时给出明确提示并打开 PDF 选择器。

7. 构建目录和可执行文件升级为 `WordPdfVoice-v0.3.2`，继续共用 v0.2 数据目录。

## 保留能力

- 提取全部 4450 条词汇和来源页码。
- 序号作为稳定唯一标识，重复拼写保持独立。
- 异常标记、搜索、编辑、逐词试听和音频缓存。
- 本地 Kokoro ONNX 美式或英式发音。
- 中断续做、失败可见和有效音频跳过。
- 导出包含真实 WAV 的 Anki `.apkg`。
- Windows 便携版，不依赖系统 Python。

## 用户流程

```text
选择 PDF
  → 首次进入 v0.2 时复制 v0.1 已有结果
  → 查看全部词条或勾选“只看已有音频”
  → 试听 / 编辑 / 继续生成
  → 可随时“导出已有音频”验证 Anki
  → 全部完成后“导出全部 Anki”
```

## 版本与数据边界

- v0.1.1 程序和 `%LOCALAPPDATA%\WordPdfVoice\projects\` 保持不变。
- v0.5.0 程序目录为 `dist\WordPdfVoice-v0.5.0\`，与 v0.2.x、v0.3.x、v0.4.0 共用 v0.2 数据目录。
- v0.2.0 数据目录为 `%LOCALAPPDATA%\WordPdfVoice\v0.2\projects\`。
- 首次迁移使用 SQLite 一致性快照；每个已完成音频复制到 v0.2 目录并重写数据库路径。
- 迁移完成后，删除或继续运行 v0.1 都不会影响 v0.2 已复制的数据。

## 验收条件

- 勾选“只看已有音频”后，只显示数据库状态为 `ready` 的词条。
- 搜索、异常筛选和已有音频筛选可组合使用。
- 部分 Anki 导出只包含实际存在的音频，不因其余词条未生成而失败。
- 没有任何有效音频时，部分导出给出明确提示，不生成空卡组。
- 全量导出仍要求全部词条有有效音频。
- v0.2 首次迁移后，数据库、人工编辑和音频路径均指向 v0.2 独立目录。
- 真实 PDF 仍提取 4450 条、序号连续，原始 PDF 哈希不变。
- 便携版必须通过真实 24 kHz 单声道 WAV 生成门禁。
- Anki 数据库中尖括号教材标记按文字保留，不被解释为 HTML 标签。
- 相同声音和语速重复试听时复用缓存；声音或语速变化后重新生成。
- v0.2.0 默认音频首次由 v0.2.1 使用时不重复生成，只补写配置。
- 序号升序为 `1、2、3…`，降序为 `4450、4449、4448…`，不能按字符串排序。
- 便携版真实生成不同声音以及 0.65、1.25 倍速 WAV，并验证声音内容和音频长度确实不同。
- 1000×680 逻辑窗口在 Windows 150% 显示缩放下完整显示，不出现横向或底部裁切。
- 每次启动的短句和贴纸都来自五枚内置原创资源与用户本地集合，并避免与上一次连续重复。
- 五枚内置贴纸可从源码与便携版加载，左上角像素透明，仓库与安装包不存在商业动漫角色贴纸。
- 从任意所选行连续播放，重复次数、暂停和中英文模式实际生效；失败单词可跳过，停止按钮能中断队列。
- 认识、模糊、不认识写入 SQLite；重点复习只返回模糊和不认识，重新打开后状态保留。
- v0.3.2 数据首次打开前生成迁移备份，新增字段后原词条、人工修改与音频索引不变。
- 备份必须通过 `quick_check`；恢复前保存当前数据库，恢复后学习状态与词条可读取。
- 构建前版本一致性检查通过，最终文件名、窗口标题和版本标签均为 v0.5.0。
- 主页面层级与确认的 v0.5.0 示例图一致；播放键为圆形，上一词和下一词无普通按钮边框。
- 更多菜单和学习设置能够访问 v0.4.0 的音频生成、Anki、编辑、备份恢复、贴纸、声音、语速、重复、暂停和中英文播放功能。
