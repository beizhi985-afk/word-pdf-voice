# 语音设置变化但仍播放旧缓存

## 现象

界面切换声音或语速后，再次点击“试听所选”，听到的声音与之前完全相同。

## 原因

旧缓存只按词条序号判断 WAV 是否已经生成，没有记录生成时使用的声音、语速和语言。只要文件存在且状态为 `ready`，程序就直接播放旧文件，新的界面设置没有机会进入语音引擎。

## 修复

- 为每条音频增加 `audio_profile`，记录 `语音引擎版本 + 声音 + 语速 + 语言`；
- 当前设置与缓存配置不同时，试听和批量生成自动重新制作 WAV；
- v0.2.0 固定使用 `af_sarah + 0.90 + en-us`，因此旧音频可按这个默认配置直接认领，不需要无意义地全部重做；
- 编辑词条、重置词条或标记音频缺失时，同时清空缓存配置；
- 便携版构建门禁真实生成不同声音、慢速和快速 WAV，比较内容哈希和音频帧数。

## 防复发规则

1. 任何会改变生成结果的参数都必须进入缓存键，不能只按输出文件是否存在判断。
2. 新增缓存字段时必须为旧数据库提供自动加列或其他向前兼容迁移。
3. 语音控件测试不仅检查“参数传入”，还要验证参数变化会使旧缓存失效。
4. 成品验证至少比较两个声音，并比较同一声音的慢速和快速音频长度。
