# 四级词汇 PDF 表格提取规则

## 现象

目标 PDF 视觉上是跨 107 页的四列表格，部分长释义在单元格内换行；按纯文本阅读顺序提取时，长释义可能跑到序号前面。

## 影响范围

如果直接使用页面纯文本，单词、注音和释义可能错位，尤其是跨行释义和长单词。

## 根本原因

PDF 的视觉阅读顺序不等于逻辑词条顺序，但表格边框和单元格结构完整。

## 解决办法

使用 `pdfplumber` 的表格提取结果，不使用整页纯文本作为主数据源。每行必须满足四列结构，第一列必须是数字序号；页眉和页脚单独忽略。

## 验证方式

- 全文应得到 4450 条。
- 序号应为 1-4450 连续集合。
- 每条必须有单词，允许注音或释义进入异常队列。
- 抽查第 1、2、54、107 页的视觉渲染与结构化结果。

## 防复发规则

1. 不得以 `page.extract_text()` 作为目标文档的正式提取路径。
2. 修改表格规则后必须运行真实 PDF 回归测试。
3. 词条唯一标识使用序号，不使用单词拼写。
4. 单元格内换行只能规范为空格，不能拆成新词条。
