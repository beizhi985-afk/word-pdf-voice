# Qt 后台任务启动后没有执行

## 现象

选择 PDF 后，界面保持响应，底部一直显示“正在分析文档…”，但没有页码进度、错误提示或项目数据库。

## 根因

窗口只保存了 `QThread`，没有保存移动到线程中的 Python `QObject` worker。信号连接不足以保证 Python 包装对象持续存活；打包环境中 worker 可能在 `thread.started` 调用其槽函数之前被垃圾回收。

## 修复

- 窗口用以 `QThread` 为键的映射强引用所有活动 worker。
- 线程结束时同时释放 worker 引用和线程引用。
- 增加生命周期回归测试：删除调用方的局部引用并强制垃圾回收后，worker 仍必须完成任务。
- PDF 分析期间禁用重复导入按钮，并即时显示正在处理的文件名。

## 防复发规则

凡是通过 `moveToThread()` 运行的 Python `QObject`，其拥有者必须保存强引用直到线程结束；不能只依赖 Qt 信号连接或局部变量维持生命周期。
