# 连续播放英文和中文静默无声

## 现象

单独生成的英文 WAV 有效，但正式 GUI 中启动连续播放后听不到英文；选择“英文＋中文释义”时中文也没有声音。原测试使用假的播放回调，只确认了循环顺序，没有覆盖 Windows 实际扬声器链路。

## 原因

1. 英文使用后台线程中的 `winsound.PlaySound`，它没有给应用提供可检查的设备状态，正式环境可能静默返回。
2. 中文由隐藏 PowerShell 进程直接调用 `SpeechSynthesizer.Speak`，与英文走完全不同的音频输出链路。
3. 中文改为生成 WAV 后，最初仍把输出路径追加在 PowerShell `-Command` 后；Windows PowerShell 5 对这类参数的解析不稳定，`SetOutputToWaveFile` 实际收到无效路径。

## 修复

- 使用 `winmm.waveOutOpen / waveOutWrite` 直接播放 PCM WAV，等待 `WHDR_DONE` 后再进入下一步。
- 中文使用系统已安装的 `zh-*` 语音先生成临时 WAV，再交给相同播放器。
- 临时路径通过 `WORD_VOICE_CHINESE_WAV` 环境变量传递，不拼接到脚本或命令行参数。
- 播放器轮询统一停止事件；用户停止时调用 `waveOutReset`，中文合成进程也会被终止。

## 防复发规则

1. 连续播放测试既要验证循环顺序，也要验证默认播放器收到同一个停止事件。
2. 正式包发布前必须实际播放一条英文 WAV 和一句中文释义，不能只验证“成功生成文件”。
3. 涉及 PowerShell 的中文路径或临时路径优先通过环境变量传递。
4. 数据格式门禁至少检查 PCM 声道数、位宽、采样率和有效帧数据。
