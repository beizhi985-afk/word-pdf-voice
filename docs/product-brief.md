# 产品定义：单词文档配音

## 项目是什么

“单词文档配音”是一款面向语言学习者的文档处理与发音学习软件。用户导入没有配音的词汇 PDF 后，软件识别其中的单词、音标、释义和例句，为每条学习内容生成对应语音，并提供逐词试听、检查和导出能力。

## 项目不是什么

- 不是只从第一页读到最后一页的普通 PDF 朗读器。
- 不是未经检查就把 OCR 结果直接批量转成音频的黑盒工具。
- 不是修改或覆盖用户原始 PDF 的编辑器。
- 首版不是通用有声书、视频配音或声音克隆平台。

## 为谁服务

主要用户是已经拥有词汇表、教材讲义或单词 PDF，但缺少配套发音资料的语言学习者。首版以个人学习和 Windows 使用场景为优先。

## 解决什么问题

目前常见做法是逐个复制单词到词典、浏览器或语音网站，再手动保存音频或制作卡片。这一过程重复、耗时，并且容易出现单词、释义和音频错配。

本项目希望做到：

- 一次导入整份文档；
- 自动理解词表而不是只读取连续文本；
- 标出识别不确定的内容，让用户集中校对；
- 批量生成与词条一一对应的发音；
- 让生成结果可以点读、重复播放和导出复用。

## 核心使用流程

```text
导入 PDF
  → 检测文本型或扫描型文档
  → 提取单词、音标、释义和例句
  → 展示结构化预览与低置信度提示
  → 用户确认或修正
  → 批量生成单词/例句语音
  → 点读学习或导出学习资料
```

## 成功标准

项目成功不只意味着“能够发出声音”，还需要满足：

1. 用户能确认语音对应哪一个词条。
2. 自动结果出错时有明显提示和修正入口。
3. 原始 PDF 始终保持不变。
4. 同一份文档可以稳定重复处理。
5. 学生无需理解 OCR、TTS 等技术概念也能完成操作。

## 当前阶段

当前处于 v0.1 可行性原型阶段。仓库结构、产品目标和约束已经建立；下一步需要用户提供一份代表性 PDF，并确认首要语言、口音、离线要求和主要输出格式，然后用真实样本验证整个流程。
