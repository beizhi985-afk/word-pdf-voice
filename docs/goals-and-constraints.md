# 项目目标与约束

## 产品目标

让语言学习者可以把没有配音的词汇文档转成可听、可核对、可反复学习的资料，尽量把人工复制单词和逐个寻找发音的过程压缩成一次导入与少量校对。

## 核心价值

- 一键导入文档，而不是逐条复制单词。
- 单词、音标、释义、例句和语音保持明确对应。
- 对识别不确定的内容给出提示，避免“看似成功但读错词”。
- 允许重复播放、调整语速，并保留可迁移的学习结果。

## 产品约束

1. **学生优先**：默认方案应尽量低成本，并避免复杂部署。
2. **Windows 优先**：首版首先保证普通 Windows 电脑可用，其他平台后续评估。
3. **原文只读**：不得覆盖、重写或破坏用户原始 PDF。
4. **隐私透明**：本地处理优先；若使用云端 OCR 或语音，必须明确显示上传内容、服务商和可能费用。
5. **结果可核对**：自动提取后必须能预览和修正，低置信度内容不能静默进入最终音频。
6. **语音可替换**：语音引擎与文档处理解耦，避免绑定单一供应商。
7. **接口稳定**：非公开或可能随时失效的免费接口不得成为唯一生产方案。
8. **版权谨慎**：仓库不提交用户教材原件、批量生成音频或无授权资源。

## 工程约束

- PDF 处理需要同时考虑文本型、扫描型和混合型文档。
- 词表通常存在多栏、页眉页脚、音标、换行例句等结构，不能只按纯文本顺序朗读。
- 语音文件必须使用稳定标识与词条绑定，避免同名词覆盖和重新生成后错配。
- 处理过程应能缓存中间结果，修改一条词条时不应重新生成全部音频。
- 关键流程需要可重复测试：提取、分组、规范化、语音生成和导出。

## 当前未知项

以下信息需要通过用户样本和选择确认，确认后更新本文件与 `spec/current.md`：

- PDF 的实际版式与扫描质量；
- 首要学习语言、口音和发音标准；
- 连续朗读与逐词点读的优先级；
- 是否需要 Anki、独立 MP3、可点读网页或其他输出；
- 离线要求、硬件条件和可接受成本。
