# 更新日志

本文件记录用户可感知的功能变化、重要修复和版本范围调整。

## [未发布]

### 新增

- 按 Obsidian 项目仓库方式增加根目录完整方案、方案索引、项目交接和中文更新日志导航。
- 完成 PDF 词表提取、异常校验、SQLite 持久化、本地语音缓存、Qt 桌面界面和 Anki 导出实现。
- 增加真实四级词汇 PDF 的结构回归测试：107 页、4450 条、序号完整。
- 成功生成 30 个代表性词的 Kokoro ONNX 本地 WAV，全部通过音频完整性检查。
- 成功打包包含 30 个实际音频的测试 Anki 卡组。
- 增加 Windows 中文路径下的 eSpeak ASCII 运行时兼容处理。
- 完成无需系统 Python 的 Windows 便携版打包；成品已通过启动检查，并内置离线语音模型。

### 修复

- 显式关闭 SQLite 连接，修复 Windows 临时数据库无法删除的问题。
- 桌面界面从依赖系统 Tcl/Tk 的 Tkinter 调整为可随应用打包的 Qt/PySide6。
- 修复打包版选择 PDF 后后台 worker 被提前回收，界面一直停在“正在分析文档…”的问题。
- 分析期间显示当前文件并禁用重复导入；支持把 PDF 路径作为启动参数直接分析。

## [0.2.0] - 2026-07-18

### 新增

- 增加“只看已有音频”，支持与搜索、异常筛选组合查看。
- 增加“打开音频文件夹”，可直接检查本地 WAV。
- 增加“导出已有音频”，未完成全部 4450 词时也能导出可验证的 Anki 卡组。
- 摘要区在试听和批量任务结束后刷新已有音频数量。

### 版本与数据

- v0.2.0 使用独立程序目录 `WordPdfVoice-v0.2.0` 和独立数据目录。
- 首次打开同一 PDF 时，通过 SQLite 一致性快照复制 v0.1 已有词条、人工修改和有效 WAV。
- 从后续版本起采用普通升级，只提升版本号并完整维护更新日志，不再默认复制整套程序和数据。

### 验证

- 新增已有音频筛选、部分 Anki 导出、空卡组拒绝和独立数据迁移测试。
- 17 项自动测试通过，真实 PDF 仍为 107 页、4450 条、序号连续。

### 修复

- 导出时转义释义中的 `<古>`、`<口>` 等教材标记，避免被 Anki 误当作 HTML 标签而隐藏。

## [0.1.1] - 2026-07-18

## [0.1.0] - 2026-07-17

### 第一版方案确认

- 第一版目标确定为“英文单词 PDF -> 逐词播放 + Anki 导出”。
- 目标文档确定为 `英语四级词汇完整版带音标-乱序版--共107页.pdf`。
- 已验证目标文档为 107 页文本型四列表格，共 4450 条词汇，序号连续且无需 OCR。
- 第一版使用序号作为词条唯一标识，避免 `present`、`forbid` 等重复拼写错误复用音频。
- 第一版采用本地优先路线：PDF 只读、SQLite 缓存、Kokoro ONNX 英语发音、Windows 桌面点读和包含实际音频的 Anki 卡组。
- 扫描件 OCR、直接修改 PDF、手机应用、例句配音和云端同步不进入第一版。
